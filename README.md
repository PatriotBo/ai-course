## 授课规则

每节课固定流程：

1. 回顾上节课；
2. 检查上次作业或豁免状态；
3. 读取课程长期规则和进度 Ledger；
4. 补充今日 AI 热门术语；
5. 输出今日备课单并正式授课；
6. 如有代码实战，补充必要注释和 Python 语法说明，但避免重复解释已经在前课讲过的通用内容；
7. 代码示例默认面向真实 provider 配置运行，除非明确要求，不再单独生成 mock provider / mock demo；
8. 布置一套课后练习/作业，不再另设重复的课堂练习；
9. 批改作业并更新进度、课程首页和历史回看中心。

说明：只有真实课堂互动、设计评审、代码调试或诊断内容明显不同于课后作业时，才单独生成课堂记录。
