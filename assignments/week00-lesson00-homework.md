### 7.1 用户提交摘要

#### 基础题

1. **AI 应用工程师和传统后端工程师最大的区别**
   传统后端的代码逻辑相对固定、输出稳定；AI 应用工程师面对的是大模型的不稳定输出，需要通过工程手段保证稳定输出。

2. **为什么会调 LLM API 不等于会做 AI 应用**
   调用 LLM API 只是其中很小的一步。AI 应用还需要处理用户输入、Prompt、工具调用、结构化输入输出、成本优化、上下文管理、缓存、监控、Trace、Eval 等多个环节。

3. **RAG / Tool Calling / Agent / MCP 的理解**
   - RAG：解决大模型缺少领域私有知识的问题；
   - Tool Calling：让大模型获得调用工具和操作外界环境的能力；
   - Agent：具备工具调用、记忆、上下文管理和计划执行能力；
   - MCP：用户初始理解为多个 Agent 交互能力，本次批改中已修正为“标准化连接外部工具、数据源和上下文资源的协议”。

### 7.3 本次重点修正

1. **MCP 不是多个 Agent 互相通信协议**
   MCP 的核心是标准化 AI 应用 / Agent 连接外部工具、数据源和上下文资源。

2. **Agent 不主要是解决“注意力分散”**
   更准确的公式是：`Agent = LLM + Tools + Memory/State + Planning + Execution Loop`。

3. **RAG 不只是补私有知识**
   RAG 更完整的作用是：让模型基于外部可追溯资料回答，降低幻觉，并支持私有知识、实时知识、引用来源和权限控制。
