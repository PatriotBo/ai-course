## 2. 用户回答摘录

### 问题 1：Harness vs LLM

用户回答：

> Harness 不是模型本身，而是为了让模型能够更好的执行任务的框架/外壳，主要涉及边界管理，权限管理，工具调用，结果观测等方面，使得任务执行更精准，不跑偏，不越界。

诊断：回答正确。已经抓住 Harness 的核心：它不是推理模型本体，而是模型外层的工程执行框架，负责工具、权限、边界和观测。

---

### 问题 2：Agent Loop 为什么需要状态管理

用户回答：

> Agent Loop 如果不管理状态，那和 chatbot 没有什么区别，只能一问一答，无法执行复杂的任务。

诊断：方向正确。需要进一步补充：状态管理不仅让 Agent 能执行复杂任务，还决定它能否恢复、续跑、限步、审计和 debug。

推荐补全表达：

```text
Agent Loop 需要状态管理，因为每一步工具调用、观察结果、计划变更、错误信息和剩余预算都要写回状态。
如果没有状态，Agent 无法知道任务进行到哪一步，也无法恢复失败、限制循环、审计过程或继续多步骤任务，最终会退化成普通 Chatbot 的单轮问答。
```

---

### 问题 3：ReAct 三段式后端类比

用户回答：

> Thought 对应请求校验，逻辑判断；Action 对应接口调用；Observation 对应结果响应。

诊断：基本正确，但 Thought 的类比还可以更精确。

更准确的对应关系：

| ReAct 阶段 | 后端类比 | 说明 |
|---|---|---|
| Thought | 决策逻辑 / 编排逻辑 / 下一步路由判断 | 判断当前任务缺什么信息、该调用哪个工具、是否继续执行 |
| Action | RPC / HTTP / DB / 搜索 / 文件操作调用 | 按 schema 调用外部工具或系统能力 |
| Observation | 下游返回值 / 工具执行结果 / 错误信息 | 写回上下文，供下一轮 Thought 使用 |

需要注意：请求校验通常属于 Harness 层，而不是 Thought 本身。Thought 更像“业务决策”和“下一步计划”。

## 3. 课堂诊断

### 已掌握

1. 能区分 LLM 和 Harness：模型负责推理，Harness 负责工程执行。
2. 能理解 Agent Loop 不只是多轮聊天，而是依赖状态推进复杂任务。
3. 能初步把 ReAct 映射到后端系统：判断、调用、响应。

### 需要加强

1. Harness 的模块化设计能力：Prompt、Tool、State、Memory、Eval、Guardrails、Trace 要拆得出来。
2. Thought 和 Harness 的边界：Thought 是模型的决策推理；校验、权限、执行、日志通常属于 Harness。
3. 从业务产品反推 Agent 运行框架：例如“吵架帮手”不只是聊天，而是训练目标、对手人格、评分、记忆、安全边界和复盘系统。

## 4. 标准补充：吵架帮手 Agent Harness 应该管理什么

如果设计“吵架帮手 Agent”，Agent Harness 至少要管理这些模块：

| 模块 | 作用 | 后端类比 |
|---|---|---|
| Prompt 管理 | 管理不同场景、人格、难度、训练目标的 Prompt 模板 | 配置中心 / 模板系统 |
| Tool 管理 | 管理评分工具、情绪检测工具、话术建议工具、历史记录查询工具 | RPC 服务注册 / Tool Router |
| State 管理 | 记录当前训练轮次、用户发言、对手状态、分数、是否结束 | 任务状态表 / 会话状态 |
| Memory 管理 | 保存用户薄弱点、常见表达问题、历史训练画像 | 用户画像 / 长期记忆 |
| Score / Eval | 对说服力、逻辑、情绪控制、攻击性、清晰度打分 | 质量评估系统 |
| Guardrails | 防止生成辱骂、威胁、人身攻击、违法建议等内容 | 风控规则 / 内容安全 |
| Trace / Log | 记录模型输入输出、工具调用、评分过程、异常 | 日志监控 / Trace |
| Budget / Limit | 限制最大轮数、token 成本、超时、工具调用次数 | 限流 / 超时 / 熔断 |
| Feedback | 生成复盘建议和下一轮训练重点 | 报告生成 / 训练反馈 |

推荐架构草图：

```text
用户选择训练场景
  ↓
Harness 加载场景 Prompt / 对手人格 / 难度配置
  ↓
Agent Loop 开始
  ↓
用户发言
  ↓
LLM 扮演对手并生成回应
  ↓
评分工具评估用户表现
  ↓
State 记录当前轮次、得分、薄弱点
  ↓
Guardrails 检查内容边界
  ↓
继续下一轮 or 生成复盘报告
```
