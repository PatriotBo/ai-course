---

## 1. 本节核心结论

真实项目选择不能只看 star 数，也不能被热门项目名字带着走。

更合理的标准是：

```text
适合当前阶段 > 技术名气 > star 数
```

一个适合学习和作品集沉淀的 AI 项目，至少要满足：

- 能跑；
- 能懂；
- 能裁剪；
- 能改造；
- 能展示；
- 能讲清楚。

---

## 2. 本节建立的方法

以后选择 AI 项目，先用 10 个维度快速打分：

| 维度 | 看什么 |
|---|---|
| 目标匹配度 | 是否贴合 AI 工程师转型目标 |
| 主链路清晰度 | 是否能画出输入到输出 |
| 可运行性 | 能否本地跑最小 demo |
| 可裁剪性 | 能否只复刻一小段能力 |
| 工程化含量 | 是否有日志、配置、错误处理、测试、评估 |
| 学习曲线 | 当前阶段能不能吃下 |
| 作品集价值 | 能否写进简历 / GitHub / X |
| 改造空间 | 能否结合吵架帮手、投研助手等方向 |
| 资料质量 | README、examples、issue、文档是否清楚 |
| 风险成本 | API、部署、数据、合规风险是否可控 |
